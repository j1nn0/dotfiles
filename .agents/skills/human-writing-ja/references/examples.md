# Before/After 具体例

文書タイプごとの改善例。文脈に応じた文体で改善している点に注目。

## 企画書（ですます調）

社内向け企画書。提案の意図と具体的な施策を伝える。

### Before

```
## 企画概要:

本企画では、顧客満足度向上のための施策についてご提案させていただきます。

### 背景:

近年、顧客ニーズは多様化しており、これにより従来のサービスでは対応が困難な状況となっています。顧客満足度を向上させることは非常に重要です。

### 施策内容:

以下の施策を実施することで、顧客満足度を向上させることができます。

- **迅速な**対応体制の構築
- **きめ細やかな**フォローアップの実施
- **継続的な**改善活動の推進

### 期待効果:

本施策を実施することで、さまざまな効果が期待できます。顧客満足度の向上により、売上増加に貢献することが可能となります。
```

### ■問題点

- 見出しがコロンで終わっている
- 「ご提案させていただきます」
- 「これにより」
- 「～することができます」
- 「非常に重要です」（理由なし）
- 太字の多用
- 「さまざまな」
- 施策が抽象的で何をするのかわからない

### After

```
## 企画概要

問い合わせ対応の待ち時間を短縮し、顧客満足度を上げる施策です。

### 背景

直近3ヶ月の顧客アンケートで「対応が遅い」という回答が32%ありました。
現状、問い合わせへの初回返信に平均26時間かかっています。

### 施策内容

1. チャットボットを導入し、よくある質問（全体の40%）を自動回答する
2. 対応担当者を2名から4名に増員する
3. 回答テンプレートを整備し、返信作成時間を短縮する

### 期待効果

初回返信を平均4時間以内にします。
導入費用は月額15万円、増員コストは月額60万円です。
```

## FAQ（ですます調）

ヘルプページやサポートサイト向け。質問に端的に答える。

### Before

```
## Q: パスワードを忘れてしまいました。どうすればよいですか？

パスワードをお忘れになった場合は、パスワードの再設定を行うことができます。これにより、新しいパスワードを設定することが可能となります。

まず、ログイン画面にアクセスしていただき、「パスワードをお忘れの方」というリンクをクリックしてください。次に、ご登録いただいているメールアドレスを入力していただきます。最後に、届いたメールに記載されているリンクから、新しいパスワードを設定していただければ、再度ログインしていただくことができます。

ご不明な点がございましたら、お気軽にサポートまでお問い合わせください。
```

### ■問題点

- 「これにより」
- 「～することができます」「～することが可能となります」
- 「まず」「次に」「最後に」の機械的な列挙
- 「いただき」の連続
- 「お気軽に」
- 回答が冗長

### After

```
## Q: パスワードを忘れました

ログイン画面の「パスワードをお忘れの方」から再設定できます。

1. 登録メールアドレスを入力
2. 届いたメールのリンクを開く
3. 新しいパスワードを設定

メールが届かない場合は、迷惑メールフォルダを確認してください。
```

## 報告書（ですます調）

社内報告書は会社の文化による。ここではですます調で改善。

### Before

```
## 月次活動報告書

### 1. はじめに

本報告書では、2024年12月度の営業活動について報告させていただきます。

### 2. 活動概要

当月は、新規顧客開拓を中心に活動を行いました。これにより、さまざまな成果を上げることができました。具体的には、以下の取り組みを実施しました。

- 展示会への出展
- Web広告の配信
- 既存顧客へのアップセル提案
- 新規営業電話の実施
- パートナー企業との連携強化

### 3. 成果

上記の活動の結果、以下のような成果を上げることができました。

- 新規問い合わせ: 45件（前月比+20%）
- 商談設定: 12件（前月比+15%）
- 受注: 3件（前月比+50%）

### 4. 課題と今後の展望

一方で、以下のような課題も明らかになりました。これらの課題に取り組むことが重要です。

- 問い合わせから商談への転換率が低い
- Web広告のコストパフォーマンスが悪い
- 既存顧客のフォローが不十分

今後は、これらの課題に取り組むとともに、さらなる成果向上を目指してまいります。
```

### ■問題点

- 「はじめに」セクションが不要
- 「報告させていただきます」
- 「これにより」
- 「さまざまな」
- 「～することができました」が2回
- 「上記の」
- 箇条書きが5項目以上
- 「これらの課題に取り組むことが重要です」（同義反復）
- 「目指してまいります」の曖昧な宣言

### After

```
## 12月度 営業報告

### 活動内容

展示会に出展し、名刺を230枚獲得しました。
Web広告は50万円の予算で配信しています。
既存顧客15社にアップセルを提案しました。

### 成果

| 指標 | 12月 | 前月比 |
|-----|-----|-------|
| 問い合わせ | 45件 | +20% |
| 商談 | 12件 | +15% |
| 受注 | 3件 | +50% |

### 課題

問い合わせの27%しか商談に至っていません。
初回対応に平均48時間かかっていることが原因と考えられます。
Web広告は問い合わせ単価1.2万円で、展示会（単価4千円）の3倍です。

### 1月の対応

問い合わせ対応を24時間以内に行うルールを設けます。
Web広告は出稿先を1媒体に絞り、予算を30万円に減らします。
```

## 製品説明（ですます調）

製品説明は顧客向けなのでですます調が基本。

### Before

```
## 製品概要:

本製品は、社内文書のデジタル化を実現する**革新的な**クラウドソリューションです。これにより、業務効率を大幅に向上させることができます。

### 特長:

本製品には、以下のような特長があります。

- **圧倒的な**処理速度
- **シームレスな**外部連携
- **直感的な**操作性
- **堅牢な**セキュリティ

### 導入効果:

本製品を導入することで、さまざまなメリットを享受することができます。多くの企業様にご導入いただいており、高い評価をいただいております。

ぜひ一度お試しください！
```

### ■問題点

- 見出しがコロンで終わっている
- 「これにより」
- 「～することができます」が2回
- 太字の多用
- 「革新的な」「圧倒的な」「シームレスな」などの大げさな形容詞
- 「さまざまな」「多くの」
- 「高い評価をいただいております」（具体性なし）
- 特長が抽象的で何ができるかわからない

```

### 導入効果

| 項目 | 導入前 | 導入後 |
|-----|-------|-------|
| 書類検索時間 | 15分 | 30秒 |
| 承認完了までの日数 | 3日 | 当日 |

### 無料トライアル
```

### Before

## 改行の問題（技術ドキュメント）

英語式の「すべての改行を段落分け（空行）」は避ける。日本語では句点ごとの改行が自然。

### Before

```
## セットアップ

まず、リポジトリをクローンしてください。次に、依存関係をインストールします。最後に、開発サーバーを起動します。

git clone https://github.com/example/repo.git
cd repo
npm install
npm run dev

ブラウザでhttp://localhost:3000を開くと、アプリケーションが表示されます。初回起動時は少し時間がかかる場合があります。
```

### ■問題点

- 文が横に長く続いている
- 手順説明なのに改行がない
- すべての改行が空行になっている

### After

```
## セットアップ

リポジトリをクローンしてください。
依存関係をインストールして、開発サーバーを起動します。

git clone https://github.com/example/repo.git
cd repo
npm install
npm run dev

ブラウザでhttp://localhost:3000を開きます。
初回起動時は少し時間がかかります。
```
